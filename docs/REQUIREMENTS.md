# Skill Management System - Requirements

## 已实现功能

### 1. 技能提交流程
- [x] 用户通过 ZIP 文件上传提交技能
- [x] 用户通过 source_url 提交技能
- [x] 从已安装技能一键提交到精选
- [x] 已提交技能自动隐藏提交按钮（避免重复提交）

### 2. AI 自动审核
- [x] 部署时可配置是否启用（BRIDGE_ENABLE_AUTO_REVIEW）
- [x] 审核任务队列（打卡机制：领取/完成）
- [x] 30分钟超时自动释放任务
- [x] 基础格式检查（SKILL.md、frontmatter）

### 3. 管理员二审
- [x] 审核列表（pending/approved/rejected 筛选）
- [x] 通过/拒绝操作
- [x] 审核备注

### 4. 通知系统
- [x] 管理员二审后发送站内通知给用户
- [x] 通知包含 AI 初审结果（评分、建议）
- [x] 通知包含管理员备注

### 5. 前端展示
- [x] 用户提交列表显示 AI 审核结果（评分、问题分类）
- [x] 技能状态显示（pending/ai_reviewed/approved/rejected）
- [x] 右上角通知铃铛（未读数、通知列表）

### 6. Agent 创建
- [x] 模型选择
- [x] 预装精选技能（创建时选择）

## 待实现功能

### 1. 隔离环境/沙盒测试
审核时除了静态分析，还应在隔离环境（Docker/Firecracker/gVisor）中实际执行技能，检测运行时安全问题。

### 2. 更健壮的 AI 审查
当前仅基础正则检查，未来应接入 LLM 进行深度语义分析、代码安全扫描。

### 3. 聊天对话中提交技能
用户在聊天中可通过自然语言指令（如"把这个技能提交到精选"）提交技能审核。

### 4. 技能版本升级审核 (Diff 模式)
技能更新时只审核改动部分，而非完整重新审核。

### 5. 批量审核功能
管理员可批量通过/拒绝多个技能提交。

### 6. 审核优先级队列
VIP 用户、紧急修复的技能优先审核。

### 7. 技能组合包
替代"禁用技能"的设计，支持预定义技能组合（如"开发工具包"），创建 Agent 时选择安装哪些组合包。

### 8. 技能商店独立服务评估
当前是 Platform Gateway 的一部分，技能数量激增（>10000）后再考虑是否独立服务。

---

## 明确不做的功能

- **AI 初审单独通知用户**: 仅站内通知列表可见，不发送推送
- **审核 Agent 主动发消息给用户主 Agent**: 避免过多消息打扰
